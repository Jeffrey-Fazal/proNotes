Using Git

Decide if you want to start a repo locally or clone/fork an existing project.

To start a local repo create it with git init

The simple stages of a git project (without branching)

  • Use git status to check what has been staged, commited and changed. It's good practie to use this before pushing
  • After you have working code you are ready to stage it. Use git add . to stage your working directory or git add /pathtofile for a fille/folder
  • YOu are findally ready to commit the project. Use git commit -m "description of commit"to add it to the repo
  • Not out of the woods yet, you'll want to push it to your remote repo with git push
  • If you're working from a fork and the code it ready then submit a pull request (PR request)

Useful git commands

Naming Conventions

Camel case always starts out lowercase with each word delimited by a capital letter (like personOne, textUtil, thingsToDo)

Pascal case is similar to camel case, but the first letter is always capitalized (like PersonOne, TextUtil, ThingsToDo)

Snake case means that we delimit words with an underscore (like person_one, text_util, things_to_do)
